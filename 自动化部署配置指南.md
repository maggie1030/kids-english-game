# 🚀 GitHub 自动化部署配置指南

配置完成后，以后所有项目都可以**自动部署**，你只需要专注写代码！

---

## 📋 配置步骤（一次性，约5分钟）

### 第一步：创建 GitHub 仓库

1. 打开 https://github.com/new
2. 填写仓库信息：
   - **Repository name**: `kids-english-game`（或任意名称）
   - **Description**: 儿童英语游戏中心
   - **Public** ✅（推荐公开，免费使用 Actions）
   - **Initialize this repository with**: ✅ Add a README
3. 点击 **Create repository**

### 第二步：上传代码到仓库

在你的终端执行：

```bash
# 进入项目目录
cd ~/Desktop/儿童英语游戏

# 初始化 Git
git init

# 添加远程仓库（替换 your-username 为你的 GitHub 用户名）
git remote add origin https://github.com/your-username/kids-english-game.git

# 添加所有文件
git add .

# 提交
git commit -m "🎮 初始化英语游戏项目"

# 推送到 GitHub
git push -u origin master
# 如果失败，尝试: git push -u origin main
```

### 第三步：添加腾讯云密钥到 GitHub Secrets

这是**最关键的一步**，密钥将安全存储在 GitHub：

1. 打开你的 GitHub 仓库页面
2. 点击顶部 **Settings** 标签
3. 左侧菜单选择 **Secrets and variables** → **Actions**
4. 点击绿色按钮 **New repository secret**
5. 依次添加以下 4 个 Secrets：

| Secret Name | 说明 | 值从哪里获取 |
|------------|------|-------------|
| `TENCENT_SECRET_ID` | 腾讯云 API ID | https://console.cloud.tencent.com/cam/capi |
| `TENCENT_SECRET_KEY` | 腾讯云 API Key | 同上，点击「新建密钥」|
| `TENCENT_BUCKET` | 存储桶名称 | 你创建的 COS 存储桶名称 |
| `TENCENT_REGION` | 存储桶地域 | 如 `ap-hongkong`、`ap-shanghai` |

**添加示例：**
```
Name: TENCENT_SECRET_ID
Secret: YOUR_SECRET_ID_HERE  (你的实际 SecretId)
```

点击 **Add secret** 保存，重复添加全部 4 个。

### 第四步：验证部署

1. 在 GitHub 仓库页面，点击 **Actions** 标签
2. 你会看到工作流正在运行（黄色圆圈）
3. 等待 1-2 分钟，变成绿色对勾 ✅ 即部署成功
4. 访问网址：`https://你的存储桶名称.cos-website.地域.myqcloud.com`

## 🔒 安全说明

- ✅ 密钥存储在 GitHub Secrets，**加密保护**
- ✅ GitHub Actions 运行时可以读取，但**不会显示在日志中**
- ✅ 其他人**无法查看**你的密钥
- ✅ 比本地存储更安全（不会误提交到代码中）

## ❓ 常见问题

### Q: Actions 运行失败怎么办？
A: 点击 Actions → 失败的运行记录 → 查看错误日志，通常是 Secrets 名称拼写错误

### Q: 可以部署到其他平台吗？
A: 可以！只需修改 deploy.yml，支持 Vercel、阿里云、AWS 等

### Q: 多个项目可以用同一套密钥吗？
A: 可以！每个项目重复添加相同的 4 个 Secrets 即可
